## Реализованная функциональность

### 1. Backend (NestJS + Prisma)

#### Архитектура базы данных
- **users** - пользователи системы
- **projects** - проекты пользователей
- **workflows** - рабочие процессы
- **workflow_nodes** - узлы в workflow (универсальная таблица)
- **workflow_edges** - связи между узлами
- **node_types** - типы узлов (registry)
- **executions** - запуски workflow
- **execution_logs** - логи выполнения узлов
- **credentials** - учётные данные (отдельно от конфигурации)
- **workflow_shares** - публичный доступ к workflow

#### API модули
- **Auth** - аутентификация и регистрация
- **Users** - управление пользователями
- **Projects** - CRUD проектов
- **Workflows** - CRUD workflow
- **Node Types** - получение типов узлов
- **Executions** - запуск и просмотр выполнений
- **Credentials** - управление секретами
- **Workflow Shares** - публичный доступ

#### Бизнес-логика
- Создание admin пользователя при старте
- Валидация workflow перед выполнением
- Логирование выполнения узлов
- Docker конфигурация для развёртывания

### 2. Frontend (React + React Flow)

#### Страницы
- **Login/Register** - аутентификация
- **Projects** - список проектов
- **Workflows** - список workflow в проекте
- **Workflow Editor** - визуальный редактор workflow
- **Settings** - настройки профиля

#### Компоненты Workflow Editor

##### FlowCanvas
- Основной canvas для работы с workflow
- Интеграция с React Flow
- Drag & Drop узлов
- Соединение узлов рёбрами

##### NodesPalette
- Палитра доступных узлов
- Категории: Triggers, Logic, Data, Actions, Integrations
- Drag & Drop узлов на canvas

##### CustomNodes
- **TriggerNode** - узлы-триггеры (manual_trigger, webhook_trigger)
- **DefaultNode** - обычные узлы (http_request, if, switch, code, etc.)
- Визуализация с иконками SVG
- Handles для соединений

##### CustomEdges
- Кастомные рёбра для соединения узлов
- Стилизация связей

##### BottomLogsPanel
- Панель логов выполнения workflow
- Отображение статуса узлов
- Просмотр результатов выполнения

#### Реализованные типы узлов

**Триггеры:**
- `manual_trigger` - ручной запуск
- `webhook_trigger` - запуск по webhook

**Логика:**
- `if` - условное ветвление
- `switch` - множественное ветвление

**Данные:**
- `set` - установка значений
- `transform` - трансформация данных

**Действия:**
- `http_request` - HTTP запросы
- `code` - выполнение кода (JavaScript/Python)
- `delay` - задержка выполнения

**База данных:**
- `db_select` - выборка из БД
- `db_insert` - вставка в БД

#### Иконки узлов
- Все узлы имеют SVG иконки
- Маппинг типов на иконки в `nodeIconMap.ts`
- Отображение названий узлов через `nodeDisplayNames`

## Архитектурные решения

### Универсальная таблица узлов
Вместо отдельных таблиц для каждого типа узла используется одна таблица `workflow_nodes` с полем `configJson` для хранения специфичных настроек.

### Разделение Design-time и Runtime
- **Design-time:** структура workflow, узлы, связи
- **Runtime:** executions, logs (отдельные таблицы)

### Отделение секретов
Credentials хранятся отдельно от конфигурации узлов для безопасности.

### Node Registry
Типы узлов хранятся в таблице `node_types` с метаданными:
- displayName
- category
- inputs/outputs
- defaultConfig
- schema
