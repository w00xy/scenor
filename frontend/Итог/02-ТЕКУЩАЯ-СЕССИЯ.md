### 2. Компонент ResizableNodeConfig

**Файл:** `ResizableNodeConfig.tsx`

**Функциональность:**
- Язычок (resizer) для изменения ширины центральной секции
- Расположение: по центру верхней части секции Параметры
- Курсор: `ew-resize` (стрелки влево-вправо)
- Drag & Drop для изменения ширины
- Ограничения: min 300px, max 1200px

**Визуальные эффекты:**
- Ручка: 4px × 30px
- Цвет по умолчанию: #555
- При наведении/перетаскивании: #ff4b33
- Плавные переходы цвета

**Условное отображение:**
- Показывается только для узлов с `hasInput={true}` и `hasOutput={true}`
- Триггеры (без входа) не имеют язычка

### 3. Конфигурации для всех типов узлов

#### Реализованные конфиги (11 компонентов):

**Триггеры:**
1. **ManualTriggerConfig** - ручной запуск (без настроек)
2. **WebhookTriggerConfig** - путь webhook, HTTP метод

**Логика:**
3. **IfConfig** - условия (field, operator, value), режим AND/OR
4. **SwitchConfig** - выражение, варианты (value, output)

**Данные:**
5. **SetConfig** - значения в формате JSON
6. **TransformConfig** - JavaScript код для трансформации

**Действия:**
7. **HttpRequestConfig** - URL, метод, headers, query, body, timeout
8. **CodeConfig** - язык (JavaScript/Python), исходный код
9. **DelayConfig** - задержка в миллисекундах

**База данных:**
10. **DbSelectConfig** - таблица, условия WHERE (JSON)
11. **DbInsertConfig** - таблица, значения (JSON)

#### Общие стили (NodeConfig.scss)

**Секции:**
- Вход/Выход: #212121, flex: 1 1 0
- Параметры: #2B2B2B, 550px (изменяемая через resizer)
- Границы между секциями: #444

**Поля ввода:**
- Input/Select/Textarea: фон #2a2a2a, граница #444
- Focus: граница #ff4b33

- Шрифт: Inter

**Кнопки:**
- Добавить условие: #2a2a2a, граница #444

### Обработка событий

**Двойной клик:**
```typescript
const handleNodeDoubleClick = useCallback((nodeId: string) => {
  setNodes((currentNodes) => {
    const node = currentNodes.find(n => n.id === nodeId);
    if (node) {
      setConfigModal({
        isOpen: true,
        nodeId: node.id,
        nodeType: node.type || 'default',
        nodeData: node.data
      });
    }
    return currentNodes;
  });
}, []);
```

**Drag & Drop resizer:**
```typescript
const handleMouseDown = (e: React.MouseEvent) => {
  const startX = e.clientX;
  const startWidth = paramsWidth;
  
  const handleMouseMove = (moveEvent: MouseEvent) => {
    const deltaX = moveEvent.clientX - startX;
    const newWidth = Math.max(300, Math.min(1200, startWidth + deltaX));
    setParamsWidth(newWidth);
  };
  
  document.addEventListener('mousemove', handleMouseMove);
  document.addEventListener('mouseup', handleMouseUp);
};
```

## Исправленные проблемы

### 1. Модальное окно не открывалось
**Проблема:** `handleNodeDoubleClick` зависел от `nodes` через `useCallback`, обработчик пересоздавался, но узлы уже были созданы со старой ссылкой.

**Решение:** Использование функциональной формы `setNodes` для получения актуального состояния без зависимостей.

### 2. Отсутствие иконки в header
**Проблема:** `NodeConfigWrapper` передавал `displayName` вместо кода типа узла.

**Решение:** Передача `nodeType` (код типа) для корректного поиска иконки через `nodeIconMap`.

### 3. Колонки не занимали всю ширину
**Проблема:** Использовались `max-width` ограничения на боковых секциях.

**Решение:** Изменение на `flex: 1 1 0` без ограничений max-width.

### 4. Ошибки сборки (незакрытые теги)
**Проблема:** При рефакторинге структуры не были закрыты div теги для секций.

**Решение:** Исправление структуры JSX во всех конфигах с добавлением закрывающих тегов.
