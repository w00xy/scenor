## ai.scenor.ru — сервис генерации сценариев для видео

ai.scenor.ru помогает маркетологам, продюсерам и авторам быстро получать сценарии для роликов: от рекламных интеграций до обучающих видео. Проект стартует с базового генератора текстов и постепенно будет расширяться новыми инструментами для полного цикла подготовки контента.

### Текущие возможности
- Создание сценариев по краткому описанию или брифу
- Гибкие шаблоны с настройкой тона, длительности и CTA
- Экспорт текста и совместный доступ к сценариям
- Система тегов для организации сценариев
- Комментарии к сценариям
- Совместное редактирование сценариев

### План развития
- Дополнительные AI‑модули (расшифровки, резюмирование)
- Командные рабочие пространства, роли и общий медиаархив

### Архитектура
- Backend: `FastAPI` (Python) для REST/WS и фоновых задач
- Frontend: `React` + `SCSS` для SPA
- Хранилище: PostgreSQL + S3-совместимый стор для ассетов
- Очереди и фоновые задачи: Celery + Redis (планируется)

### API Документация
API автоматически документирован с использованием стандартов OpenAPI/Swagger. После запуска приложения документация будет доступна по адресу:
- `/docs` - интерактивная документация Swagger UI
- `/redoc` - документация ReDoc
- `/openapi.json` - спецификация OpenAPI в JSON формате

### API Эндпоинты
- `/auth/` - аутентификация и регистрация пользователей
- `/scenarios/` - управление сценариями
- `/templates/` - шаблоны сценариев
- `/ai/generate/` - генерация сценариев через ИИ
- `/history/` - история диалогов
- `/tags/` - теги для сценариев
- `/comments/` - комментарии к сценариям
- `/scenario-shares/` - совместное редактирование

### Локальный запуск
1. Установите зависимости: `pip install -r backend/requirements.txt`
2. Настройте переменные окружения в файле `.env`
3. Запустите миграции: `alembic upgrade head`
4. Запустите приложение: `python main.py`

### Запуск с Docker
1. Установите Docker и Docker Compose
2. Создайте файл `.env` с необходимыми переменными окружения:
   ```
   OPENAI_API_KEY=your_openai_api_key
   ```
3. Запустите сервисы: `docker-compose up -d`
4. Выполните миграции: `docker-compose exec backend alembic upgrade head`

### Структура проекта
- `backend/` - основной код бэкенда
- `backend/app/` - исходный код приложения
- `backend/app/api/v1/` - API маршруты
- `backend/app/models/` - модели базы данных
- `backend/app/schemas/` - Pydantic схемы
- `backend/app/services/` - бизнес-логика
- `backend/app/repositories/` - Repository паттерн
- `backend/app/observers/` - Observer паттерн
- `backend/migrations/` - миграции базы данных
- `backend/tests/` - тесты
