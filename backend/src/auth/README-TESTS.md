# Автоматизированное тестирование JWT токенов

## Обзор

Создан полный набор автоматизированных тестов для проверки JWT токенов на бекенде.

## Созданные тесты

### 1. AuthTokenService Tests (`auth-token.service.spec.ts`)
**16 тестов** - Проверка генерации и валидации JWT токенов

#### Генерация токенов:
- ✓ Генерация access и refresh токенов для пользователя
- ✓ Генерация токенов для SUPER_ADMIN роли
- ✓ Обработка отсутствия JWT_ACCESS_SECRET
- ✓ Обработка отсутствия JWT_REFRESH_SECRET
- ✓ Использование дефолтных значений времени истечения

#### Проверка access токенов:
- ✓ Валидация корректного access токена
- ✓ Отклонение невалидного токена
- ✓ Отклонение истекшего токена
- ✓ Отклонение refresh токена вместо access
- ✓ Проверка токена с SUPER_ADMIN ролью
- ✓ Обработка отсутствия секрета

#### Проверка refresh токенов:
- ✓ Валидация корректного refresh токена
- ✓ Отклонение невалидного токена
- ✓ Отклонение access токена вместо refresh
- ✓ Проверка токена с SUPER_ADMIN ролью
- ✓ Обработка отсутствия секрета

### 2. AuthGuard Tests (`auth.guard.spec.ts`)
**12 тестов** - Проверка guard для защиты роутов

#### Валидация заголовков:
- ✓ Доступ с валидным Bearer токеном (USER)
- ✓ Доступ с валидным Bearer токеном (SUPER_ADMIN)
- ✓ Отклонение при отсутствии заголовка Authorization
- ✓ Отклонение заголовка без префикса Bearer
- ✓ Отклонение пустого токена после Bearer
- ✓ Отклонение токена из пробелов
- ✓ Обработка токена с лишними пробелами
- ✓ Отклонение при ошибке верификации
- ✓ Отклонение истекшего токена
- ✓ Отклонение некорректного формата Bearer
- ✓ Проверка чувствительности к регистру
- ✓ Обработка множественных пробелов

### 3. RolesGuard Tests (`roles.guard.spec.ts`)
**13 тестов** - Проверка контроля доступа на основе ролей

#### Контроль доступа:
- ✓ Доступ когда роли не требуются
- ✓ Доступ когда массив ролей пустой
- ✓ Доступ для USER роли
- ✓ Доступ для SUPER_ADMIN роли
- ✓ Доступ при совпадении одной из ролей
- ✓ Доступ SUPER_ADMIN при множественных ролях
- ✓ Отклонение при отсутствии требуемой роли
- ✓ Отклонение при undefined роли
- ✓ Отклонение при отсутствии объекта user
- ✓ Отклонение USER на SUPER_ADMIN роуте
- ✓ Отклонение при несовпадении ролей
- ✓ Корректный вызов reflector
- ✓ Обработка null ролей

### 4. Упрощенные тесты

#### `auth-token.service.simple.spec.ts` - 8 тестов
Базовые тесты генерации и проверки токенов на русском языке

#### `auth.guard.simple.spec.ts` - 6 тестов
Базовые тесты авторизации на русском языке

### 5. E2E тесты (`auth-jwt.e2e-spec.ts`)
Интеграционные тесты для полного flow аутентификации (требуют настройки окружения)

## Запуск тестов

### Все auth тесты:
```bash
npm test -- auth-token.service.spec.ts auth.guard.spec.ts roles.guard.spec.ts
```

### Упрощенные тесты:
```bash
npm test -- auth-token.service.simple.spec.ts auth.guard.simple.spec.ts
```

### Все тесты с фильтром:
```bash
npm test -- --testNamePattern="auth"
```

### E2E тесты:
```bash
npm run test:e2e -- auth-jwt.e2e-spec.ts
```

## Покрытие

Тесты покрывают:
- Генерацию JWT токенов (access и refresh)
- Валидацию токенов
- Проверку типов токенов
- Обработку ошибок и edge cases
- Авторизацию через Bearer токены
- Контроль доступа на основе ролей (RBAC)
- Обработку некорректных заголовков
- Проверку секретных ключей из переменных окружения
